# TradeVault — راهنمای فارسی

**[English](README.md)** | **[فارسی](README_FA.md)**

**ژورنال حرفه‌ای معاملات با ردیابی سود و زیان، تحلیل استراتژی، منحنی سرمایه، تقویم حرارتی، نمودار TradingView و ردیابی چالش پراپ فرم.**

این برنامه در دو نسخه موجود است:
- **نسخه ابری** (`worjer.js`) — Cloudflare Worker + D1 + KV، همگام‌سازی چند دستگاهی
- **نسخه محلی** (`index.html`) — فایل HTML مستقل، ذخیره در مرورگر، بدون نیاز به نصب

## امکانات

### ژورنال معاملات
- ثبت معاملات با نماد، جهت (خرید/فروش)، قیمت ورود، خروج، حد ضرر، حد سود، حجم
- محاسبه خودکار سود/زیان و R-multiple بر اساس نوع ابزار (فارکس، کریپتو، فلزات، شاخص‌ها)
- انتخاب زمان ورود و سشن معاملاتی (پیش‌فرض: زمان فعلی، تشخیص خودکار سشن)
- ردیابی سودهای جزئی (TP1، TP2، TP3 با تیک‌باکس)
- MAE/MFE (بیشترین حرکت نامطلوب/مطلوب)
- امتیازدهی معامله (۱ تا ۵ ستاره)
- سیستم تگ (افزودن تگ‌های دلخواه، فیلتر بر اساس تگ)
- فیلدهای سفارشی (تعریف فیلدهای خودتان برای هر حساب)
- تا ۴ تصویر برای هر معامله، هر کدام با یک یادداشت
- قفل ۴۸ ساعته معاملات (اعمال شده در سرور — معاملات بعد از ۴۸ ساعت فقط‌خواندنی می‌شوند)

### تحلیل‌ها
- منحنی سرمایه (SVG، سبز بالای موجودی شروع، قرمز زیر آن)
- تقویم حرارتی (نمایش بصری سود/زیان روزانه)
- نرخ برد، ضریب سود، شارپ، انتظاری، نسبت پرداخت
- محاسبه حداکثر افت سرمایه
- رشته‌های پیروزی/شکست متوالی
- نمودار میله‌ای سود ماهانه
- تحلیل عملکرد بر اساس روز هفته
- جدول تفکیک نمادها
- ردیابی بهترین/بدترین معامله

### ردیاب پراپ فرم
- ردیابی چالش‌های FTMO، FundedNext و سایر پراپ فرم‌ها
- انواع حساب: یک‌مرحله‌ای، دو مرحله‌ای، فاند شده
- ردیابی حد زیان روزانه و کل
- نوار پیشرفت هدف مرحله
- تشخیص خودکار قبولی مرحله ۱
- **مودال گزارش به سبک MT5** شامل:
  - منحنی سرمایه (از موجودی شروع، تقسیم سبز/قرمز)
  - ۱۲ کارت آماری عملکرد
  - وضعیت حساب (موجودی، رشد، رشته‌های متوالی، استفاده از حد زیان)
  - نمودار سود ماهانه
  - نمودار و جدول عملکرد روزهای هفته
  - جدول تفکیک نمادها
  - جدول تاریخچه معاملات

### مدیریت استراتژی
- ساخت استراتژی با چک‌لیست، یادداشت و URL ویدیو
- اتصال معاملات به استراتژی‌ها از طریق منوی کشویی
- تحلیل استراتژی (نرخ برد، سود به ازای هر استراتژی)

### فرصت‌های معاملاتی
- ثبت فرصت‌های معاملاتی با نماد، یادداشت و تصویر چارت
- برچسب‌گذاری به عنوان خرید/فروش
- نمایش اسلایدشویی تصاویر چارت

### یکپارچه‌سازی نمودار
- ویجت نمودار پیشرفته TradingView (تایم‌فریم ۱۵ دقیقه پیش‌فرض)
- اندیکاتور RSI
- نوار جانبی واچ‌لیست همگام با واچ‌لیست شما
- تشخیص خودکار پیشوند نماد (OANDA, BINANCE, TVC, INDEX)
- همگام‌سازی تم تیره/روشن

### تقویم اقتصادی
- دریافت تقویم اقتصادی هفتگی از سرور
- امکان آپلود دستی فایل CSV
- فیلتر بر اساس سطح تاثیر (زیاد، متوسط، کم، همه)
- تبدیل منطقه زمانی

### واچ‌لیست
- افزودن/حذف نماد
- دسته‌بندی از پیش تعریف شده (جفت‌های اصلی فارکس، کریپتو، شاخص‌ها، فلزات)
- قابلیت جابجایی با درگ و دراپ
- همگام‌سازی با نوار جانبی TradingView

### یادداشت‌های بازبینی
- کاردهای خلاصه خودکار هفتگی و ماهانه
- یادداشت‌های قابل ویرایش برای هر دوره
- ذخیره به ازای هر حساب

### پشتیبان‌گیری و بازیابی
- پشتیبان‌گیری کامل (تمام حساب‌ها، تمام داده‌ها) در فایل JSON
- بازیابی از JSON (داده‌های فعلی را پاک کرده و جایگزین می‌کند)
- سازگار بین دو نسخه (پشتیبان ابری → محلی و برعکس)
- حفظ شناسه‌ها (ID) در طول فرآیند پشتیبان‌گیری/بازیابی

## شروع سریع

### گزینه ۱: نسخه محلی (بدون نیاز به نصب)

۱. فایل `index.html` را دانلود کنید
۲. آن را در هر مرورگر مدرنی باز کنید
۳. شروع به ثبت معاملات کنید — تمام داده‌ها در localStorage مرورگر ذخیره می‌شوند

بدون سرور، بدون ثبت‌نام، بدون نیاز به اینترنت. داده‌ها در مرورگر شما می‌مانند.

### گزینه ۲: نسخه ابری (چند دستگاهی)

۱. به [داشبورد Cloudflare](https://dash.cloudflare.com) بروید → Workers & Pages
۲. یک Worker جدید بسازید
۳. محتوای `worjer.js` را در آن قرار دهید
۴. متغیرهای محیطی را اضافه کنید (راهنمای کامل در [DEPLOY_GUIDE](DEPLOY_GUIDE.md))
۵. پایگاه داده D1 و فضای KV بسازید
۶. ذخیره و Deploy کنید
۷. آدرس Worker خود را باز کنید، رمز عبور را وارد کنید

## دو نسخه برنامه

| ویژگی | نسخه ابری | نسخه محلی |
|---|---|---|
| فایل | `worjer.js` | `index.html` |
| سرور | Cloudflare Worker + D1 | localStorage مرورگر |
| احراز هویت | رمز عبور (KV sessions) | ندارد |
| چند دستگاهی | بله | خیر |
| آفلاین | خیر | بله |
| محدودیت ذخیره‌سازی | سهمیه رایگان D1 | حدود ۵-۱۰ مگابایت |
| فرمت پشتیبان | Cloud JSON (v4) | Local JSON (کلیدهای tv_*) |
| سازگاری متقابل | بله | بله |

### انتقال بین نسخه‌ها

**محلی → ابری:**
۱. نسخه محلی را باز کنید → پشتیبان‌گیری → فایل JSON دانلود می‌شود
۲. نسخه ابری را Deploy کنید
۳. برنامه ابری را باز کنید → پشتیبان‌گیری → بازیابی → فایل را انتخاب کنید

**اکری → محلی:**
۱. برنامه ابری را باز کنید → پشتیبان‌گیری → فایل JSON دانلود می‌شود
۲. نسخه محلی را باز کنید → پشتیبان‌گیری → بازیابی → فایل را انتخاب کنید

هر دو نسخه به‌طور خودکار فرمت پشتیبان را تشخیص داده و در صورت نیاز تبدیل می‌کنند.

## امنیت

- **احراز هویت:** مبتنی بر رمز عبور با توکن نشست KV (TTL ۳۰ روزه، تمدید خودکار)
- **محدودیت نرخ:** ۱۰ تلاش ناموفق ورود به ازای هر IP در ۵ دقیقه
- **جلوگیری از SQL Injection:** تمام کوئری‌ها از پارامترهای مقید استفاده می‌کنند
- **محافظت IDOR:** هر UPDATE/DELETE بر اساس account_id محدود می‌شود
- **قفل ۴۸ ساعته:** اعمال شده در سرور — معاملات قدیمی‌تر از ۴۸ ساعت قابل ویرایش یا حذف نیستند
- **XSS:** تمام ورودی‌های کاربر قبل از نمایش escape می‌شوند
- **CORS:** قابل پیکربندی از طریق متغیر محیطی `ALLOWED_ORIGIN`

## پشتیبان‌گیری و بازیابی

### ایجاد پشتیبان

۱. روی دکمه **Backup** در برنامه کلیک کنید
۲. یک فایل JSON با تمام داده‌های شما دانلود می‌شود

### بازیابی پشتیبان

۱. روی **Restore** در برنامه کلیک کنید
۲. فایل JSON پشتیبان خود را انتخاب کنید
۳. تایید کنید — این کار **تمام داده‌های فعلی را پاک کرده** و با پشتیبان جایگزین می‌کند
۴. صفحه با داده‌های بازیابی شده بارگذاری مجدد می‌شود

### انتقال بین نسخه‌ها

پشتیبان‌های هر دو نسخه در هر دو نسخه قابل بازیابی هستند:
- پشتیبان ابری → برنامه محلی: تبدیل خودکار به کلیدهای tv_* در localStorage
- پشتیبان محلی → برنامه ابری: تبدیل خودکار به فرمت ابری، ارسال به `/api/restore`

## تکنولوژی‌ها

- **سرور:** Cloudflare Workers (ESM), D1 (SQLite), KV
- **فرانت‌اند:** HTML/CSS/JS خالص (بدون فریم‌ورک، بدون build)
- **نمودارها:** TradingView Advanced Chart, SVG برای منحنی سرمایه
- **ذخیره‌سازی:** D1 (ابری) یا localStorage (محلی)
- **احراز هویت:** رمز عبور + توکن نشست KV
- **تست:** Node.js + node:sqlite (شبیه‌سازی D1)، بیش از ۷۴,۰۰۰ assertion
